# 読書withCoco Content Guidelines

この文書は、レビュー文、SNS投稿、ギャラリー文、画像プロンプト、CTA文言を作るときの制作ガイドラインである。ブランド原典は `docs/brand/reading-with-coco-brand-strategy.md` を参照する。

## 制作原則

- 投稿単体で価値を渡す。サイト誘導やアフィリエイトは補助にする。
- 本の要約ではなく、読者の生活に残る問い、変化、行動を作る。
- 難解な概念は、読者の日常語に翻訳する。
- 読者を急かさない。煽らない。見下さない。
- ココちゃんは、難しい知への入口をやわらかくする案内役として使う。

## 基本構成

投稿やレビューは、必要に応じて以下の順で組み立てる。

1. 読者のモヤモヤ: 仕事、家庭、体調、役割変化などの具体的な違和感から始める
2. 本の問い: その本が読者に投げかける問いを示す
3. 見方の変化: 読む前と後で、世界の見え方がどう変わるかを書く
4. 実生活への一歩: 今日できる小さな行動に落とす
5. ココちゃんの入口: 難しい内容をやわらかく受け止める視点を添える

すべてを毎回入れる必要はない。ただし、最後に「問い」か「行動」が残ることは必須である。

## 4つの投稿軸

### 1. 問い

- 読者の内側に残る問いを1つに絞る
- 正解を押しつけず、読者が自分の生活へ引き寄せられる形にする
- 例: 「今の自分は、何を守るために疲れているのか」

### 2. 変化

- 読む前と後の見え方の差を描く
- 大げさな人生逆転ではなく、視界が少し整う変化を扱う
- 例: 「不安を消す本」ではなく「不安との距離を測り直す本」

### 3. 実生活

- 今日できる小さな行動を示す
- 抽象論で終わらず、会話、休息、仕事、家事、メモなどの場面に落とす
- 例: 「寝る前に、今日の違和感を1行だけ書く」

### 4. ココちゃん

- 難しい知を受け止めやすくする入口を作る
- 本から生まれた景色を案内する
- かわいさだけでなく、読者が安心して考え始められる余白を作る

## レビュー文の判断基準

- 本の概要だけで終わっていないか
- 著者の主張を、読者の生活に接続できているか
- 読者の悩みやモヤモヤが入口にあるか
- 「誰に向いているか」が煽りではなく、生活場面で説明されているか
- 読後の行動が1つでも見えるか

## SNS投稿の判断基準

- 1投稿だけで、問いか気づきが残るか
- サイトや購入リンクへの誘導が主役になっていないか
- 短さのために、読者への寄り添いを削っていないか
- 専門用語を使う場合、すぐに日常語へ翻訳しているか
- ココちゃんの視点が、知識の入口として機能しているか

## 画像プロンプトの判断基準

- 本、ココちゃん、学びの3要素が見えるか
- ココちゃんが単なるマスコット配置になっていないか
- 本から広がる景色、読後の余韻、日常への接続があるか
- 過剰に幼い、派手すぎる、広告っぽい方向へ寄っていないか
- 既存の design doctrine と衝突しない静けさ、好奇心、余韻があるか

## Webサイト改修の判断基準

- `docs/reading-with-coco-design-doctrine.md` と併用する
- first viewport では `読書 with Coco` の brand、世界観、主要CTA を弱めない
- 機能一覧やKPI風の整理より、読後の余韻が深まる順番を優先する
- CTA は読書体験への導線を主役にし、購入やアフィリエイトだけを前面化しない
- ココちゃん関連ビジュアルを、ブランドの主役級アセットとして扱う

## アフィリエイトと導線

アフィリエイト導線は否定しない。ただし、読書withCocoでは投稿単体の価値提供を先に置く。

- よい状態: 投稿を読んだだけで、読者に問い、見方の変化、今日の一歩が残る
- 避ける状態: 投稿が購入リンクへの前置きになっている
- 判断基準: リンクを外しても、その投稿は読者の役に立つか

## 禁止事項

- 「読まないと損」「人生が変わる」と煽る
- 難しい言葉を、難しいまま並べる
- 読者の不安を刺激してクリックへ誘導する
- 本の要約だけで終える
- 書き手の感想だけで閉じる
- ココちゃんを余白埋めの飾りにする
- 投稿の主役をアフィリエイト、ランキング、価格訴求にする

## 7つの知性チェック

- [ ] 思想: この本や投稿が信じていることを示せている
- [ ] ビジョン: 読後に見える景色を描けている
- [ ] 志: 誰の何を楽にするかがある
- [ ] 戦略: SNSとサイトの役割が整理されている
- [ ] 戦術: 今日できる行動がある
- [ ] 技術: 情報設計、AI、Web表現が適切である
- [ ] 人間力: 読者に寄り添う姿勢がある
