# DESIGN.md — 読書 with Coco

本ドキュメントは、サイト全体の見た目の一貫性を保つための**実装時判断基準**です。
大企業向けデザインシステムではなく、個人開発サイトが迷わずに使える最小構成です。

---

## 1. 目的

- 新しいページやコンポーネントを作るとき、色・文字・余白の選択で迷わないようにする
- 「なんとなく今の気分で」ではなく、サイトの世界観に合うかを客観的に判断する
- AI 実装時にもこのファイルを参照すれば、ブランドを壊さずに実装できるようにする

---

## 2. ブランド原則

> サイトの核は「本 × ココちゃん × 学び」

| 原則               | 具体的な意味                                                        |
| ------------------- | ------------------------------------------------------------------- |
| やわらかさ          | 角丸は大きめ、影は薄く、色は暖色を静かに使う                        |
| 清潔感              | 余白をたっぷり取り、情報密度は文字組みで整える                      |
| 落ち着き            | アニメーションは控えめ（0.2〜0.3秒）。派手な動きや強いガラス表現は避ける |
| 親しみやすさ        | 本文/UI はサンセリフ中心、ブランド強調や引用だけセリフ体を使う      |
| 物語性              | 情報の羅列ではなく、場面転換としてセクションをつなぐ                |

**NG パターン**: generic SaaS 風、テンプレ量産型 LP 風、過剰にポップ、幼児向けアプリ風

## 3. カラールール

デザイントークンは `src/styles/theme.css` の `:root` が正本です。

### 3.1 色の役割

| 役割         | 変数名                    | 値                          | 使いどころ               |
| ------------ | ------------------------- | --------------------------- | ------------------------ |
| 背景         | `--color-background`      | `#faf6ef`（温かみのある白） | ページ全体の背景         |
| 表面         | `--color-surface`         | `#fffaf3`                   | カード・セクション背景   |
| 白（強い面） | `--color-surface-strong`  | `#ffffff`                   | モーダルや強調面         |
| テキスト     | `--color-text`            | `#20160f`（濃い茶）         | 本文・見出し             |
| 控えめ       | `--color-muted`           | `#6d5846`                   | キャプション・補足       |
| アクセント   | `--color-accent`          | `#8d5b2a`                   | CTA ボタン・リンク       |
| アクセント淡 | `--color-accent-soft`     | `#f3dfb4`                   | バッジ・タグ背景         |
| 境界線       | `--color-border`          | `rgba(99,66,32,0.14)`       | カードの枠線             |
| 境界線（強） | `--color-border-strong`   | `rgba(99,66,32,0.24)`       | 区切り線・アクティブ状態 |
| フォーカス   | `--color-focus`           | `#0a5fcc`                   | フォーカスリング         |
| Hero（深）   | `--color-hero-deep`       | `#091832`                   | Hero 背景（濃）          |
| Hero（中）   | `--color-hero-mid`        | `#16325e`                   | Hero 背景（中）          |

### 3.2 色の原則

- **暖色ベース**: 茶系・ベージュ系を軸にする。寒色は Hero や特別な場面でのみ使う
- **新しい色を足すとき**: 既存の変数で表現できないか検討してから。役割が被る色を増やさない
- **色の 3 割ルール**: 背景系 70% / テキスト・アクセント 30% のバランスを保つ

## 4. タイポグラフィ

### 4.1 フォントファミリ

| 役割   | 変数名           | フォント                                         |
| ------ | ---------------- | ------------------------------------------------- |
| ブランド / 強い見出し | `--font-display` | Iowan Old Style → Palatino → Hiragino Mincho → Yu Mincho（セリフ系） |
| 見出し | `--font-headline` | Iowan Old Style → Palatino → Hiragino Mincho → Yu Mincho（要所のみセリフ系） |
| 本文 / UI | `--font-body` / `--font-ui` | -apple-system / BlinkMacSystemFont / Hiragino Sans / Noto Sans JP（サンセリフ系） |

- 本文と UI はサンセリフ中心で、Apple 的な精密さと読みやすさを担う
- ブランド名、hero の核となる見出し、引用などはセリフ体で本の温かみを残す
- 明朝を全面に敷くのではなく、セリフ体は「効かせたい場所だけ」に使う

### 4.2 サイズスケール

| 役割   | 変数名            | 値                              | 行間   |
| ------ | ----------------- | ------------------------------- | ------ |
| Display | `--text-display` | `clamp(2.8rem, 6vw, 5.2rem)`   | 1.04   |
| 見出し | `--text-title` / `--text-headline` | `clamp(2.15rem, 4vw, 3.6rem)` / `clamp(1.7rem, 3vw, 2.6rem)` | 1.12   |
| リード | `--text-lead` | `clamp(1.02rem, 1.4vw, 1.14rem)` | 1.78 |
| 本文   | `--text-body`     | `1rem`                          | 1.78    |
| キャプション | `--text-caption` | `0.82rem`                | 1.6〜  |

### 4.3 タイポグラフィの原則

- ** clamp を使う**: サイズは固定値ではなく `clamp()` で流動的にする
- **行間はゆったり**: 本文とリードは `1.78` 前後、見出しは `1.04〜1.12`
- **letter-spacing**: 大見出しはやや詰める。ラベルだけを広めに取り、本文は不自然に広げない
- **行長を制御する**: hero lead はおおむね `28〜32rem`、通常本文は `36〜40rem` を目安にする
- **改行任せにしない**: hero の補助文は長文化させず、冗長な説明は section 以降へ逃がす

## 5. 余白ルール

### 5.1 スペーススケール

| 変数名       | 値        | 使いどころ                 |
| ------------ | --------- | -------------------------- |
| `--space-1`  | `0.375rem` | 微調整・アイコンギャップ   |
| `--space-2`  | `0.625rem` | 小要素の内側余白           |
| `--space-3`  | `0.875rem` | 隣接要素の間               |
| `--space-4`  | `1rem`     | コンポーネント内パディング |
| `--space-5`  | `1.5rem`   | カード内パディング         |
| `--space-6`  | `2rem`     | セクション内ギャップ       |
| `--space-7`  | `2.5rem`   | セクション間               |
| `--space-8`  | `3rem`     | 大きなセクション間         |

### 5.2 セクション間の余白

- セクション padding は `clamp(1.5rem, 4vw, 3rem)` 程度で流動的に
- 固定値ではなく `clamp()` を基本とする
- 「詰め込みすぎ」より「余白が少し多いかな」くらいが正解

## 6. 角丸・線・影

### 6.1 角丸

| 変数名         | 値         | 使いどころ         |
| -------------- | ---------- | ------------------ |
| `--radius-sm`  | `0.875rem` | 小さいカード・タグ |
| `--radius-md`  | `1.25rem`  | 一般的なカード     |
| `--radius-lg`  | `1.75rem`  | 大きなカード・Hero |
| `--radius-pill` | `999px`   | ボタン・バッジ     |

- 角丸は**大きめ**がこのサイトらしい。`0` や `4px` は避ける
- 画像コンテナも `clamp(18px, 2vw, 24px)` 程度の角丸をつける

### 6.2 線（ボーダー）

- 基本: `1px solid var(--color-border)`（透過茶色）
- 区切りを強調したいときだけ `--color-border-strong` に上げる
- 太い線・二重線は使わない

### 6.3 影

| 変数名           | 値                                         | 使いどころ   |
| ---------------- | ------------------------------------------ | ------------ |
| `--shadow-soft`  | `0 18px 40px rgba(83,55,28,0.08)`         | カード通常   |
| `--shadow-medium`| `0 24px 56px rgba(2,8,23,0.18)`           | モーダル・Hero |

- 影は**暖色系の薄い影**（茶系 rgba）を使う。黒ベタの影は避ける
- Hover 時は少しだけ浮かせる。常時強いシャドウや強い blur は避ける
- Apple 的に参考にするのは「静かな面」と「密度の整理」であり、白黒無彩色化ではない

## 7. 画像ルール

### 7.1 アスペクト比

| 用途         | 比率        | 備考                 |
| ------------ | ----------- | -------------------- |
| Hero シーン  | `1 / 1.08`  | 縦長ポートレート     |
| 書籍カード   | `0.88 / 1`  | やや縦長             |
| チャプター   | `1.24 / 1`  | 横長                 |
| トレイル     | `4 / 3`     | 横長                 |
| ポストカード | `1 / 1`     | 正方形               |

### 7.2 画像の原則

- `object-fit: cover` を基本とする
- 画像にも角丸をつける（コンテナと同じ radius）
- 背景フォールバック色は `rgba(63, 42, 25, 0.08)` 程度の暖色
- ココちゃん関連ビジュアルは最重要アセットとして扱う

## 8. 主要コンポーネント

### 8.1 コンテナ

- 最大幅: `1120px`
- 主要レイアウトの基準幅は `theme.css` の `--container-max` を優先する
- 左右パディング: モバイル `0.25rem` 〜 デスクトップ `0.75rem`

### 8.2 カード

- 背景: 半透明の白（`rgba(255,255,255,0.72)` 〜 `rgba(255,252,248,0.94)`）
- 枠線: `1px solid rgba(100,76,42,0.12〜0.14)`
- ホバー: `translateY(-2px〜-3px)` + 影を強める
- トランジション: `0.22s ease`
- **カードは一覧整理に必要な場面だけで使う。安易に量産しない**

### 8.3 ボタン

- Primary: グラデーション（`#8f5b2f` → `#ae7443`）、白文字、pill 型
- Primary: ほぼ黒に近い暖色ニュートラルを基準にし、強い色主張は避ける
- Secondary: 半透明背景 + 薄い枠線で静かに置く
- 最小タップ高: `48px`
- パディング: `0.85rem 1.35rem`

### 8.4 バッジ・タグ

- pill 型（`radius: 999px`）
- パディング: `0.35rem 0.75rem`
- 背景は `--color-accent-soft` や半透明の暖色

## 9. ページ別ルール

### 9.1 Home（index）

- **物語の表紙**として設計。機能一覧にしない
- Hero は brand name + 世界観 + 主要 CTA を一画面で伝える
- 参考にするのは Apple のような「静かな hero 密度」であり、派手な装飾競争にはしない
- CTA 優先度: 主 = `レビューを見る`、副 = `ギャラリーを見る`
- セクションは narrative（物語のつながり）で構成する

### 9.2 Gallery

- ココちゃんが本の世界を旅する情景を見せる
- 画像が主役。テキストは補助に徹する
- 詳細ページでは余韻を残す余白を意識する

### 9.3 Reviews

- 読書の余韻・学びを伝える
- 書籍情報とレビュー本文の分離を明確に
- 購入リンク（CTA）は静かに、でも届きやすい位置に置く

### 9.4 About / Profile

- ココちゃんと管理人の関係を温かく伝える
- 長文になりすぎず、本質を短く
- 「やわらかさ」と「親しみやすさ」を最も強く出してよいページ

## 10. モバイル優先ルール

### 10.1 ブレークポイント

| 名称     | 境界      | 想定           |
| -------- | --------- | -------------- |
| Mobile   | `< 640px` | スマホ         |
| Tablet   | `640px〜` | タブレット     |
| Desktop  | `960px〜` | PC             |
| Wide     | `1140px〜`| 大画面PC       |

### 10.2 モバイル原則

- **Mobile First**: 小画面で設計し、広い画面に向けて拡張する
- フォントサイズは `clamp()` で流動的に
- グリッドは小画面で 1 カラム、広い画面で複数カラムに
- タップターゲットは最低 `44px`（ボタンは `48px`）
- ナビゲーションはモバイルでコンパクトに
- モバイルで世界観が崩れないことを必ず確認する

## 11. 実装ルール

- デザイントークンの正本は `src/styles/theme.css` の `:root`
- トークン変更は theme.css で行い、各ページでのハードコードを避ける
- nav / section label / hero lead / button の基礎ルールはできるだけ共通トークンへ寄せる
- 新しい色やサイズを足すときは、既存の変数で表現できないか先に検討する
- トランジションは `0.22s ease` を基本とする
- `clamp()` を使って流動的なサイズ・余白を保つ
- コンポーネント分割より、まずは typography / color / spacing を決める
- hover / motion は視線誘導と物語の切り替わりに限定する

## 12. AI 実装時のルール

- このファイル（DESIGN.md）を参照してデザイン判断を行うこと
- 迷ったら「読書 with Coco らしいか？」で判断する（整っているか、ではない）
- 新しい色・フォント・間隔を追加する前に、既存の変数で代用できないか確認する
- 実装前に `visual thesis` / `content plan` / `interaction thesis` を短く定義する
- 「本 × ココちゃん × 学び」の 3 要素のうち 1 つでも欠ける構成にしない
- セクションは情報の塊ではなく、場面転換として扱う
- 実コピー・実画像をダミーより優先する
